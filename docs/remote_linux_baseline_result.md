# Результат проверки удалённого Linux-стенда для baseline CDC

## Назначение проверки

Цель проверки — подтвердить работоспособность удалённого стенда для экспериментов с RP2040 как USB-устройством. Проверялась возможность сборки прошивки в WSL, удалённого перевода RP2040 в режим BOOTSEL через ESP32-C3/nRF24, прошивки UF2 на Linux-хосте, последующей загрузки экспериментальной прошивки и сбора USB-артефактов Linux.

## Проверенный сценарий

Проверенный кейс: `000_baseline_cdc`.

Последовательность:
1. В WSL генерировалась конфигурация кейса `000_baseline_cdc`.
2. В Docker выполнялась сборка `usb_case_demo.uf2`.
3. UF2-файл передавался на удалённый Linux-хост по SSH/SCP.
4. Через ESP32-C3 отправлялась RF-команда `bootloader`.
5. RP2040 переходил в режим USB BOOTSEL/UF2.
6. Linux-хост видел устройство `2e8a:0003 Raspberry Pi RP2 Boot` и том `RPI-RP2`.
7. UF2-файл копировался на `RPI-RP2`.
8. RP2040 перезагружался в экспериментальную прошивку.
9. Linux-хост видел устройство `2e8a:000a Raspberry Pi Pico`, интерфейс CDC ACM и `/dev/ttyACM0`.
10. Через CDC выполнялись команды `help`, `info`, `ping`.
11. Сохранялись артефакты `lsusb`, `lsusb -v`, `lsblk`, `journalctl -k`, список tty-устройств.

## Подтверждённые признаки успешной работы

В результате проверки подтверждено:

- RP2040 определяется Linux-хостом как `2e8a:000a Raspberry Pi Pico`;
- интерфейсы CDC привязываются к драйверу `cdc_acm`;
- создаётся устройство `/dev/ttyACM0`;
- прошивка `usb_case_demo` отвечает на команды `help`, `info`, `ping`;
- RF-подсистема прошивки активна (`rf_ready=1`);
- RF-канал настроен на `RF_CH=0x28`;
- удалённая перепрошивка через RF-команду `bootloader` работает без физического доступа к кнопке BOOTSEL.

## Особенность Linux-хоста

На Linux-хосте была обнаружена политика USB-авторизации: устройства RP2040 появлялись в `lsusb`, но ядро сообщало `Device is not authorized for usage`. Из-за этого не создавались ни mass-storage устройство `RPI-RP2`, ни `/dev/ttyACM0`.

Проблема была устранена авторизацией устройств Raspberry Pi с VID `2e8a` через sysfs и udev-правилом для PID `0003` и `000a`.

## Вывод

Удалённый стенд признан работоспособным для baseline CDC-эксперимента. Он позволяет выполнять полный цикл: сборка прошивки, удалённый перевод RP2040 в BOOTSEL, прошивка на Linux-хосте, проверка runtime-поведения и сбор USB-логов Linux.
