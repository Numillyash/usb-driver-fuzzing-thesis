# План практической части: безопасные эксперименты с USB-дескрипторами

## 1. Цель практики

Сформировать воспроизводимый слой экспериментов для оценки устойчивости USB-стека Windows и Linux к контролируемым и безопасным аномалиям на этапе дескрипции и перечисления (enumeration) устройства.

## 2. Объект и предмет исследования

- Объект: подсистема USB в целевых ОС (Windows, Linux).
- Предмет: реакции драйверного стека и механизмов PnP/инициализации на вариации USB-дескрипторов.

## 3. Экспериментальный стенд

- Управляемое USB-устройство: RP2040 (Waveshare RP2040 Zero).
- Роль RP2040: генерация штатных и аномальных, но безопасных, вариантов дескрипторов.
- Целевые хосты:
  - Windows (актуальная поддерживаемая версия, фиксируется в протоколе запуска);
  - Linux (дистрибутив и версия ядра фиксируются в протоколе запуска).

## 4. Границы безопасности

В практической части допускаются только безопасные эксперименты уровня descriptor/enumeration. Не допускаются сценарии, связанные с автоматическим выполнением команд на хосте, эскалацией привилегий, закреплением в системе и иными вредоносными полезными нагрузками.

## 5. Основные группы сценариев

1. Базовые контрольные профили (baseline):
- CDC;
- HID без генерации пользовательского ввода;
- композитное CDC+HID.

2. Аномалии дескрипторов:
- device descriptor (длина, класс, идентификаторы);
- configuration descriptor (длина конфигурации, согласованность числа интерфейсов);
- interface/endpoint descriptor (валидность классов и полей endpoint);
- string descriptor (длина, индексы).

## 6. Наблюдаемые артефакты

- Системные журналы ОС.
- USB-трейсы перечисления.
- Результаты распознавания устройства в пользовательских и системных утилитах.
- Признаки деградации: отказ инициализации, повторные попытки конфигурации, нестабильные переподключения, предупреждения драйверного уровня.

## 7. Выходные результаты

- Формализованный каталог кейсов (`experiments/cases/*.json`).
- Унифицированный шаблон фиксации наблюдений (`experiments/templates/result_template.md`).
- Материалы для межплатформенного сравнения реакций Windows и Linux.
- Основа для последующей количественной классификации устойчивости USB-стека.
